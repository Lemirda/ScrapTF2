# Demo сайта для main.py

Стильный одностраничный сайт для презентации возможностей скрипта `main.py`. Оптимизирован под GitHub Pages, с тёмной/светлой темой, плавными анимациями и стеклянными карточками.

## Структура
```
site/
├─ index.html            # Главная страница
├─ assets/
│  ├─ css/
│  │  └─ styles.css      # Стили (темы, сетки, карточки, анимации)
│  └─ js/
│     └─ main.js         # Переключение темы, копирование команд, скролл, год в футере
└─ README.md             # Это описание
```

## Что внутри
- Хедер с брендингом, навигацией и переключателем темы
- Герой‑блок с CTA‑кнопками (Скачать, Репозиторий, Демо)
- Секция «Возможности» с карточками
- Секция «Демонстрация» с примером CLI‑запуска `main.py`
- Секция «Установка и запуск» с пошаговой инструкцией
- Секция «Полезные ссылки» с карточками ссылок
- Футер с годом (подставляется автоматически)

## Как кастомизировать
Откройте `site/index.html` и замените помеченные комментариями `TODO` ссылки и тексты:
- Кнопки в герое (Скачать, Репозиторий)
- Кнопки в блоке «Демонстрация» (Документация, Issues)
- Карточки в «Полезных ссылках» и ссылки в футере
- Текст описания `main.py` и пример команды

При желании можно добавить новые карточки/секции — достаточно скопировать существующие блоки.

## Предпросмотр локально
- Просто откройте `site/index.html` двойным кликом в браузере.
- Либо поднимите простой сервер, например:
  - Python: `python -m http.server` (и открыть `http://localhost:8000/site/`)

## Деплой на GitHub Pages
Вариант с отдельной веткой (как вы и планируете):
1. Создайте ветку `gh-pages` (или любую другую для Pages).
2. Перенесите содержимое папки `site` в корень этой ветки (именно файлы, не папку `site`).
3. В настройках репозитория GitHub включите Pages и выберите источник: Branch = `gh-pages`, Folder = `/ (root)`.
4. Дождитесь публикации и получите URL.

Альтернативы:
- Настроить Pages на ветку `main` и папку `/site` (если подходит вашему флоу).

## Требования
- Статический хостинг (GitHub Pages) — никаких серверных завязок
- Современный браузер для корректной работы CSS‑эффектов
